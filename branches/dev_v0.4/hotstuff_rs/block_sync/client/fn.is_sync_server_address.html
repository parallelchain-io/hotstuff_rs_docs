Returns whether a given verifying key is recognised as a valid sync server address.

A replica is allowed to act as a sync server if either:

  1. It is a member of the current committed validator set, or
  2. One of the current speculative blocks proposes to add the replica to the validator set.

Recognising only committed and candidate validators as potential sync servers is an effective, though rather conservative solution to the problem of sybil attacks.
